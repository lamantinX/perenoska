При написании кода следовать [standard-principles.md](/.instructions/standard-principles.md).

После написания или изменения кода ОБЯЗАТЕЛЬНО:
  1. Запустить код
  2. Проанализировать вывод (не просто "запустил и пошёл дальше")
  3. Если вывод "всё ок" или "0 ошибок" — проверить на конкретном примере, что результат действительно корректен
  4. Если результат не соответствует ожиданиям — исправить и повторить проверку

**Запрещено:**
- Считать задачу выполненной без проверки результата
- Доверять выводу "успех" без верификации на реальных данных

Валидация (только Python):
  - `/principles-validate` - валидация принципов
