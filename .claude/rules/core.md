**Приоритет скиллов:** Перед использованием низкоуровневых инструментов (Bash, Edit, Write, rm, mkdir) проверить: есть ли в списке скиллов подходящий скилл для этой задачи? Если да — использовать скилл.

**SSOT:** Если в документе есть ссылка `**SSOT:** [файл](путь)` — прочитать файл перед выполнением. Нельзя работать по памяти или предположениям.

**Исполнение скиллов:** При вызове скилла или чтении SSOT ОБЯЗАТЕЛЬНО выполнять ВСЕ шаги SSOT-инструкции последовательно. ЗАПРЕЩЕНО: пропускать шаги, объединять шаги, заменять шаги прямым Write/Edit. Каждый шаг = отдельное действие с проверкой результата. Если скилл вызывает скрипт создания (create-*-file.py) — использовать его, а не Write. Если SSOT требует валидацию — запустить валидацию.

**Строгое следование SSOT:** При исполнении любого шага chain ЗАПРЕЩЕНО:
  - Придумывать свой формат вызова агентов — использовать формат из AGENT.md
  - Добавлять свои ограничения, не описанные в SSOT (например "НЕ коммить")
  - Передавать свободный текст агенту, если SSOT определяет структурированный формат
  - Импровизировать порядок, пропускать или объединять шаги
  - Работать "по памяти" при отсутствии ссылки на SSOT — сначала найти SSOT

**Шаблоны:** При создании файлов найти шаблон/пример в SSOT-инструкции и скопировать его структуру. Запрещено придумывать свой формат.

**Блокирующее подтверждение:** Если в воркфлоу скилла есть шаг с "подтверждением" — остановиться и ждать ответа пользователя.

**Принципы кода:** При написании кода следовать [standard-principles.md](/.instructions/standard-principles.md).

**Структура проекта:** При создании/изменении/удалении папок или README.md — СТОП. ЗАПРЕЩЕНО: mkdir, rm -rf, Write для README.md напрямую. Использовать скиллы:
  - `/structure-create` - создание папки (включая .instructions/)
  - `/structure-modify` - изменение папки
  - `/structure-validate` - валидация структуры

**Черновики:** После создания или изменения черновика в `/.claude/drafts/` — запустить `/draft-validate` для проверки соответствия стандарту.

**Поиск по описаниям:** При поиске документов, скриптов или скиллов по описанию — использовать `/list-search` или напрямую:
  - `python .instructions/.scripts/search-docs.py --search "запрос"` — поиск по всей документации
  - `python .instructions/.scripts/search-docs.py --type skill --search "запрос"` — поиск по конкретному типу

При работе с GitHub ОБЯЗАТЕЛЬНО использовать скиллы:
  - `/issue-create` - правильное создание Issue
  - `/issue-modify` - правильное изменение Issue (обновление, закрытие, переоткрытие)
  - `/issue-validate` - валидация Issue по стандарту
  - `/branch-create` - правильное отведение Branch

При работе с хотфиксами ОБЯЗАТЕЛЬНО использовать скилл:
  - `/hotfix` - процесс хотфикса (диагностика, impact analysis, исправление)
