Помоги спланировать и реализовать новую фичу для проекта perenoska.

Аргументы: $ARGUMENTS

Порядок работы:
1. Уточни у меня требования, если они неполные
2. Определи какие файлы затронет изменение (routes, services, clients, static/app.js)
3. Напиши план реализации — список файлов и изменений
4. Начни с тестов (test-first), используй агент `tester`
5. Реализуй изменения, делегируй по агентам:
   - backend логика → агент `backend-dev`
   - маркетплейс API → агент `marketplace-integration`
   - UI изменения → агент `ui-designer`
6. Запусти `/verify` после завершения
