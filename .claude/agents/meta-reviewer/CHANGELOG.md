# CHANGELOG — meta-reviewer

История изменений стандартов анализа агента meta-reviewer.

## v1.4 (2026-02-12)

### Добавлено
- Секция «Антигаллюцинации»: точность > количество, без комплиментов, без домысливания
- Формализация P-уровней как блокирующие критерии: P1=BLOCK, P2=WARNING, P3=NOTE

---

## v1.3 (2026-02-01)

**Финальные исправления по четвёртой итерации**

### Добавлено
- Чек-лист "Проверка качества собственного отчёта" (метрики эффективности)

### Изменено
- Устранено дублирование в списке "Что искать" (категория "Двойственность")
- Контекст SSOT: только прямые ссылки, рекурсивное чтение не требуется
- Таймаут блокировки: уточнено "30 сек суммарно от первой проверки"

### Статус
**Документ готов к продакшену.** Критичных проблем: 0.

---

## v1.2 (2026-02-01)

**Исправления по третьей итерации самоанализа**

### Добавлено
- Секция "План исправлений" в формат вывода (последовательность + зависимости)
- Валидация результата после сохранения (Шаг 2)
- Правило определения типа документа (по пути, префиксу, AskUserQuestion)
- Расширен список размытых фраз ("когда требуется", "по желанию", "можно", "желательно", "рекомендуется")

### Изменено
- Алгоритм блокировки: убрано "6 проверок", оставлен явный таймаут 30 сек
- Уточнено определение "объекта" (предметная область, О ЧЁМ файл)
- Правила логирования: только анализируемый файл и SSOT-ссылки, не технические чтения

---

## v1.1 (2026-02-01)

**Исправления по второй итерации самоанализа**

### Добавлено
- Секция "6. Другие проблемы" для проблем вне 5 основных категорий
- Метрики покрытия анализа (всего секций / проанализировано / покрытие)
- Нумерация пунктов в разделе 4 (4.1, 4.2, ...)
- Обработка ошибки при записи файла (снять блокировку, вернуть ошибку)

### Изменено
- Блокировка: 6 проверок по 5 сек (30 сек), затем **вернуть ошибку** (не продолжать)
- Cleanup блокировки выполняется ВСЕГДА, даже при ошибке записи
- "Включить ВСЕ проблемы" → "семантические проблемы в рамках 5 категорий + другие"
- Логировать ВСЕ чтения (включая SSOT-ссылки и стандарты)
- Разделены шаги "Сохранить полный анализ" и "Вернуть резюме в чат"
- Уточнён формат промежуточных результатов для файлов > 5000 строк

---

## v1.0 (2026-02-01)

**Первая стабильная версия**

### Критерии анализа
- 5 категорий: Двойственность, Риск интерпретации, Ручные операции, Полнота, Слепые зоны

### Добавлено
- Полный алгоритм работы со state (блокировки, логирование всех действий)
- Секция "Обработка ошибок" с таблицей ситуаций
- Секция "Граничные случаи" для edge cases
- Секция "Метрики качества анализа" с чеклистом
- Критерий полноты анализа
- Легенда приоритетов P1/P2/P3
- Пример анализа с конкретной проблемой
- Версионирование в формате вывода
- Уточнение термина "объект" (что описывается в документе)
- Таблица стандартов для чтения по типу документа

### Изменено
- "ВСЕГДА сохранять" → сохранять И вернуть резюме в чат
- Логировать ВСЕ действия (read/write/create), не только запись
- Блокировка: проверять каждые 5 сек, через 30 сек сообщить LLM
