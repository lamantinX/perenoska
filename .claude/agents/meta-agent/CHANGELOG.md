## v1.1 (2026-02-01)

**Исправления по анализу meta-reviewer**

### Добавлено
- Секция "Зависимости между инструкциями" (standard → validation → create → modify)
- Секция "Порядок проверок перед выполнением" (5 шагов)
- Секция "Dry-run режим" для предварительного просмотра
- Секция "Batch-операции" для создания набора инструкций
- Скилл `draft-validate` в frontmatter
- Уточнение формата AskUserQuestion для подтверждений
- Уточнение, какие файлы логировать / не логировать
- Ветвление в примере (если пользователь отклонил)
- Формат CHANGELOG в секции "Версионирование"

### Изменено
- Контекст SSOT: читать прямые ссылки + 1 уровень вглубь
- Блокировка: 6 проверок × 5 сек, затем ошибка (не ждать пользователя)
- Приоритет скиллов: явно указано, что Edit/Write — НЕ fallback
- Шаг 5 цикла: уточнено, какие ошибки считаются
- Валидация не прошла: оставить файл для ручной правки
- Определение типа: добавлен пример сообщения AskUserQuestion
- finished_at: обновлять только при финальном отчёте
- Вывод: всегда в чат, драфт только для >20 ошибок

### Ограничения
- Добавлено: "НЕ создавать standard-*.md"
- Добавлено: "ВСЕГДА проверять зависимости"
