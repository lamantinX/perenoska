## Обновление

### Шаг 1: Найти скрипт

```
{область}/.instructions/.scripts/{name}.py
```

### Шаг 2: Обновить код

Внести изменения в логику.

**Обратная совместимость — сохранять:**
- Существующие аргументы командной строки
- Формат вывода (если используется программно)
- Exit codes

**Можно изменять:**
- Внутреннюю реализацию
- Добавлять новые опциональные аргументы
- Улучшать сообщения об ошибках

**Соблюдать принципы** из [standard-principles.md](./standard-principles.md).

**Для скриптов валидации** (`validate-*.py`):
- Проверить наличие `ERROR_CODES` — см. [standard-script.md § ERROR_CODES](./standard-script.md#error_codes-для-валидаторов)
- При добавлении новых проверок — добавить коды в `ERROR_CODES`

**Проверка принципов:** [validation-principles.md](./validation-principles.md).

### Шаг 3: Обновить docstring

Если изменился интерфейс — обновить секции:
- Использование
- Примеры
- Возвращает

### Шаг 4: Обновить инструкции

> **ОБЯЗАТЕЛЬНО:** если функционал скрипта изменился — обновить все инструкции, которые его используют.

**Найти инструкции:**
```bash
grep -r "{script-name}.py" --include="*.md" .
```

**Что обновлять в инструкции:**

| Изменение в скрипте | Что обновить в инструкции |
|---------------------|---------------------------|
| Новый аргумент | Примеры использования в шагах |
| Изменённый вывод | Описание ожидаемого результата |
| Изменённое назначение | Столбец "Назначение" в секции "Скрипты" |

**После обновления скрипта** — вызвать скилл:
```
/instruction-modify {путь к инструкции}
```

**Контекст для скилла:**
- Изменён скрипт `{script}.py`
- Изменение: `{описание изменения}`
- Обновить в инструкции: `{что обновить — примеры, назначение, шаги}`

### Шаг 5: Проверить совместимость с зависимостями

> **ОБЯЗАТЕЛЬНО** перед валидацией — убедиться, что все страницы, ссылающиеся на скрипт, продолжат работать.

**Найти все ссылки:**
```bash
grep -r "{script-name}.py" --include="*.md" .
```

**Для каждой ссылающейся страницы проверить:**

| Проверка | Описание |
|----------|----------|
| Аргументы | Все используемые аргументы по-прежнему работают |
| Вывод | Формат вывода совместим с ожиданиями |
| Exit codes | Коды возврата не изменились |

**Если обнаружена несовместимость:** Доработать скрипт для обратной совместимости. При необходимости изменить ссылающуюся страницу.

### Шаг 6: Проверить принципы

> **Валидация:** [validation-principles.md](./validation-principles.md)

```bash
python .instructions/.scripts/validate-script.py --principles {path}
```

**При обнаружении нарушений:**
1. Исправить код
2. Повторить проверку

### Шаг 7: Валидация формата

```bash
python .instructions/.scripts/validate-script.py {path}
```

### Шаг 8: Протестировать

```
1. Старые сценарии работают
2. Новые сценарии работают
```

### Шаг 9: Отчёт о проделанной работе

```
## 📋 Отчёт об изменении скрипта

✏️ **Изменён скрипт:** `{путь}`

🏷️ **Тип изменения:** {Обновление/Рефакторинг}

📝 **Что изменено:**
- {список изменений}

**Проверка совместимости:**
- Ссылающиеся страницы: {количество}
- Статус: совместимо ✅ / обновлены ссылки ✅

**Связанные инструкции обновлены:** {список или "не требовалось"}

✅ **Валидация:** пройдена
```

## Рефакторинг

**Когда рефакторить:**
- Скрипт > 300 строк
- Дублирование кода
- Сложная логика без разделения
- Отсутствуют разделители секций
- Нарушены принципы (KISS, DRY, YAGNI)

**Паттерны:**

| Проблема | Решение |
|----------|---------|
| Длинный скрипт | Разделить на функции |
| Дублирование | Вынести в общую функцию |
| Сложный main() | Вынести логику в отдельные функции |
| Много констант | Сгруппировать в секцию "Константы" |
| Тяжёлые зависимости | Заменить на стандартную библиотеку |

**Шаги:**

```
1. Добавить разделители секций
2. Сгруппировать константы
3. Выделить общие функции
4. Разделить логику — одна функция = одна задача
5. Протестировать все сценарии
6. Валидировать формат
```

## Удаление

### Шаг 1: Найти зависимости

```bash
python .instructions/.scripts/find-references.py {путь-к-скрипту}
```

**Если скрипт недоступен:** использовать паттерны из секции [Поиск зависимостей](#поиск-зависимостей).

### Шаг 2: Обновить зависимости

- Удалить вызовы скрипта
- Заменить на альтернативу
- Обновить документацию

### Шаг 3: Удалить из инструкций

Убрать строку из секции "Скрипты" всех связанных инструкций:

```bash
# Найти инструкции
grep -r "{script-name}.py" --include="*.md" .

# Для каждой инструкции вызвать
/instruction-modify {путь к инструкции}
```

### Шаг 4: Удалить файл

```bash
git rm {path/to/script.py}
```

### Шаг 5: Отчёт (удаление)

```
## 📋 Отчёт об удалении скрипта

🗑️ **Удалён скрипт:** `{путь}`

**Причина:** {причина удаления}

**Обновлены документы:**
- {список документов, из которых удалены ссылки}

**Статус:** удалён ✅
```

## Поиск зависимостей

### Где искать

| Место | Что искать |
|-------|------------|
| Инструкции (секция "Скрипты") | `| [{name}.py]` |
| README.md | Секция "Скрипты" |
| SKILL.md | Шаги воркфлоу |
| Другие скрипты | subprocess вызовы |

### Команды поиска

```bash
# Поиск в markdown
grep -r "script-name" --include="*.md"

# Поиск в Python
grep -r "script-name" --include="*.py"

# Поиск в скиллах
grep -r "script-name" .claude/skills/
```

## Примеры

### Обновление: добавление флага

```bash
# 1. Найти: /.instructions/.scripts/validate-script.py
# 2. Добавить флаг --json для JSON-вывода
# 3. Обновить docstring: секции Использование и Примеры

# 4. Найти инструкции, использующие скрипт
grep -r "validate-script.py" --include="*.md" .
# Найдено: validation-script.md, create-script.md

# 5. Обновить инструкции (добавить пример с --json)
# Если функционал влияет на использование — вызвать:
# /instruction-modify validation-script.md

# 6. Протестировать оба формата вывода
# 7. Проверить принципы
python .instructions/.scripts/validate-script.py --principles .instructions/.scripts/validate-script.py
# 8. Валидация формата
python .instructions/.scripts/validate-script.py .instructions/.scripts/validate-script.py
```

### Удаление: устаревший скрипт

```bash
# 1. Найти зависимости
grep -r "old-script" --include="*.md"
# Найдено: create-foo.md, modify-foo.md

# 2. Обновить: удалить вызовы, обновить документацию

# 3. Удалить из секции "Скрипты" каждой инструкции
# /instruction-modify create-foo.md
# /instruction-modify modify-foo.md

# 4. git rm .instructions/.scripts/old-script.py
```
