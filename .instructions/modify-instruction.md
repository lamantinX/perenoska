## Принципы

> **Шаблоны — из примеров SSOT.** При изменении файлов использовать шаблоны из секции "Примеры". Запрещено придумывать свой формат.

> **Строгий порядок: standard → validation → create → modify.** При создании недостающего типа — следовать порядку. Параллельное создание запрещено. Изменение порядка невозможно.

> **Режим Agent (опционально).** Изменение инструкций типов validation, create, modify МОЖЕТ выполняться агентом под контролем основного LLM.

## Обновление

**Когда обновлять:**
- Изменились правила или примеры
- Найдены неточности или устаревшая информация
- Добавляются новые шаги или секции

### Шаг 1: Найти инструкцию

```
{область}/.instructions/{name}.md
```

### Шаг 2: Проанализировать инструкцию

**Проверить на соответствие стандарту:**
- Frontmatter заполнен корректно
- Структура соответствует типу (standard/create/modify/validation)
- Все секции присутствуют и непустые

**Сформировать предложения по доработке:**

| Что проверить | Возможные проблемы |
|---------------|-------------------|
| Актуальность | Устаревшие примеры, ссылки |
| Полнота | Отсутствующие шаги, секции |
| Ясность | Непонятные формулировки |
| Скрипты | Шаги без автоматизации |

**Предложить пользователю:** список найденных проблем и рекомендации по исправлению.

### Шаг 3: Проверить связанные документы

**Проверить таблицу "Связанные документы":**
- Все 4 типа указаны (Стандарт, Валидация, Создание, Модификация)
- Ссылки валидны (файлы существуют)

**Проверить ссылки внутри инструкции:**
- Все ссылки на другие инструкции валидны
- Якоря существуют

**Для проверки ссылок:**
```
/links-validate {путь к инструкции}
```

### Шаг 4: Внести изменения

Обновить правила, примеры, ссылки по результатам анализа.

**Если изменяется standard-* инструкция:**

> **ОБЯЗАТЕЛЬНО** — после внесения изменений запустить семантический анализ.

Запустить агента meta-reviewer:
```
Task tool:
  subagent_type: meta-reviewer
  prompt: "Проанализируй файл: {путь к инструкции}"
```

**Результат анализа:**
- Если P1-проблемы найдены → исправить перед подтверждением (Шаг 4)
- Если только P2/P3 → рекомендуется исправить
- Если проблем нет → продолжить

**Если изменились шаги, которые автоматизируют скрипты:**

| Изменение в инструкции | Действие |
|------------------------|----------|
| Удалён шаг со скриптом | Удалить скрипт из секции "Скрипты" |
| Изменился шаг | Обновить "Назначение" скрипта |
| Добавлен шаг | Добавить скрипт → `/script-create` |

**Скиллы для работы со скриптами:**

| Скилл | Назначение |
|-------|------------|
| [/script-create](/.claude/skills/script-create/SKILL.md) | Создание нового скрипта |
| [/script-modify](/.claude/skills/script-modify/SKILL.md) | Обновление, рефакторинг, удаление |
| [/script-validate](/.claude/skills/script-validate/SKILL.md) | Валидация формата скрипта |

**Запросить подтверждение:**

> **ОБЯЗАТЕЛЬНО** использовать `AskUserQuestion` tool с кликабельными опциями.

**Вопрос:** "Изменения внесены. Проверьте и подтвердите."

**Опции:**
1. Подтверждаю — изменения корректны
2. Нужны правки — укажу что исправить

### Шаг 5: Обновить связанные документы

> **Выполнять только после подтверждения Шага 4.**

- README области (если изменилось описание)
- Связанные инструкции (если изменились ссылки)
- Скиллы (если изменился функционал)
- **Rules** (если изменились скиллы области) → `/rule-modify`

### Шаг 6: Валидация

```bash
python .instructions/.scripts/validate-instruction.py {path}
```

### Шаг 7: Отчёт о проделанной работе

```
## 📋 Отчёт об изменении инструкции

✏️ **Изменена инструкция:** `{путь}`

🏷️ **Тип изменения:** Обновление

📝 **Что изменено:**
- {список изменений}

📊 **Текущее состояние:**
- Описание: {description из frontmatter}
- Валидация: пройдена ✅
```

## Деактивация

> **Инструкции НЕ удаляются. Они деактивируются.**

**Когда деактивировать:**
- Инструкция больше не актуальна
- Технология/практика не используется
- Инструкция заменена другой

**Шаги:**

### Шаг 1: Найти все ссылки

```bash
python .instructions/.scripts/find-references.py {путь-к-инструкции}
```

**Если скрипт недоступен:** использовать паттерны из секции [Обновление ссылок](#обновление-ссылок).

### Шаг 2: Добавить предупреждение

После frontmatter добавить:

```markdown
> **ВАЖНО:** Эта инструкция не используется в проекте.
>
> Содержимое закомментировано. Для реактивации — раскомментировать.
```

### Шаг 3: Закомментировать содержимое

```markdown
---
description: {оригинальное описание}
standard: .instructions/standard-instruction.md
index: {область}/.instructions/README.md
---

> **ВАЖНО:** Эта инструкция не используется в проекте.
>
> Содержимое закомментировано. Для реактивации — раскомментировать.

<!--
# Оригинальный заголовок

{Оригинальное содержимое}
-->
```

### Шаг 4: Обновить ссылающиеся документы

- Удалить из related
- Пометить в README
- Обновить/деактивировать скиллы → `/skill-modify`
- **Обновить rules** (удалить ссылки на скилл) → `/rule-modify`

### Шаг 5: Отчёт о проделанной работе

```
## 📋 Отчёт о деактивации инструкции

🚫 **Деактивирована инструкция:** `{путь}`

🏷️ **Тип изменения:** Деактивация

**Причина:** {причина деактивации}

**Обновлены документы:**
- {список обновлённых ссылающихся документов}

**Статус:** деактивирована ✅
```

## Миграция

**Когда мигрировать:**
- Переименование файла
- Перемещение в другую папку
- Реструктуризация области

**Шаги:**

### Шаг 1: Найти все ссылки на старый путь

```bash
python .instructions/.scripts/find-references.py {старый-путь}
```

**Если скрипт недоступен:** использовать паттерны из секции [Обновление ссылок](#обновление-ссылок).

### Шаг 2: Создать новый файл

Копировать содержимое в новый путь.

### Шаг 3: Обновить frontmatter

Обновить `index` и другие относительные пути.

### Шаг 4: Обновить все ссылки

```bash
python .instructions/.scripts/update-references.py {старый-путь} {новый-путь}
# Или с предпросмотром:
python .instructions/.scripts/update-references.py --dry-run {старый-путь} {новый-путь}
```

**Если скрипт недоступен:** заменить старый путь на новый во всех документах вручную.

**Обновить rules:** если скилл мигрирован, обновить пути в rules → `/rule-modify`

### Шаг 5: Удалить старый файл

Только после проверки всех ссылок.

### Шаг 6: Валидация (миграция)

```bash
python .instructions/.scripts/validate-instruction.py {path}
```

### Шаг 7: Отчёт (миграция)

```
## 📋 Отчёт о миграции инструкции

✏️ **Мигрирована инструкция:**
- Старый путь: `{старый путь}`
- Новый путь: `{новый путь}`

🏷️ **Тип изменения:** Миграция

**Обновлены ссылки в:**
- {список документов с обновлёнными ссылками}

✅ **Валидация:** пройдена
```

## Обновление ссылок

### Где искать

| Место | Glob-паттерн |
|-------|--------------|
| Инструкции | `**/.instructions/**/*.md` |
| Скиллы | `/.claude/skills/*/SKILL.md` |
| README | `**/README.md` |
| Агенты | `/.claude/agents/*.md` |
| Драфты | `/.claude/drafts/*.md` |

### Типы ссылок

**Markdown-ссылка:**
```
[текст](путь/к/инструкции.md)
```

**Таблица README:**
```
| [имя.md](./путь/имя.md) | описание |
```

### Команды поиска

```bash
# Поиск в markdown
grep -r "instruction-name.md" --include="*.md"

# Поиск в скиллах
grep -r "instruction-name" .claude/skills/
```

### Действия при обнаружении

| Тип ссылки | При деактивации | При миграции |
|------------|-----------------|--------------|
| Markdown | Удалить/пометить | Обновить путь |
| README таблица | Пометить неактивной | Обновить путь |
| Скилл | Добавить пометку | Обновить путь |

## Примеры

### Обновление: добавление правила

```bash
# 1. Найти: /.instructions/standard-api.md
# 2. Зона: /src/**/api/**
# 3. Добавить правило версионирования
# 4. Проверить API-файлы на соответствие
# 5. Обновить README, связанные инструкции
# 6. Валидация
python .instructions/.scripts/validate-instruction.py .instructions/standard-api.md
```

### Деактивация: устаревшая инструкция

```bash
# 1. Найти: /.instructions/standard-legacy-api.md
# 2. Найти ссылки: grep -r "standard-legacy-api.md" --include="*.md"
# 3. Добавить предупреждение, закомментировать содержимое
# 4. Обновить ссылающиеся документы
```

### Миграция: переименование

```bash
# 1. Найти ссылки: grep -r "naming.md" --include="*.md"
# 2. Копировать: naming.md → naming-conventions.md
# 3. Обновить frontmatter
# 4. Заменить ссылки: naming.md → naming-conventions.md
# 5. Удалить naming.md
# 6. Валидация
python .instructions/.scripts/validate-instruction.py .instructions/naming-conventions.md
```
