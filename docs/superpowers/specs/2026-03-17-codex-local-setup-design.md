# Codex Local Setup Design

**Date:** 2026-03-17

## Goal

Подготовить репозиторий для локальной работы через Codex: нормализовать читаемость текстовых файлов, добавить локальные инструкции и компактную документацию по проекту, а также зафиксировать базовый workflow запуска и проверки без навязывания командных процессов.

## Scope

В работу входят:

- нормализация UTF-8 и читаемости ключевых текстовых файлов;
- обновление `README.md` и при необходимости метаданных проекта;
- добавление локального `AGENTS.md`;
- добавление папки `docs/codex/` с обзором проекта, архитектурой, file map, workflow и backlog notes;
- добавление `.editorconfig` и `.gitattributes`;
- привязка локального git workflow к форку пользователя `https://github.com/lamantinX/perenoska`;
- при уместности добавление `scripts/verify.ps1` и короткой шпаргалки по командам.

В работу не входят:

- изменения продуктовой логики приложения;
- рефакторинг backend/frontend вне подготовки среды;
- навязывание общекомандных процессов или CI-политик.

## Current Project Snapshot

Проект представляет собой FastAPI MVP для переноса карточек товаров между Wildberries и Ozon. В репозитории уже есть:

- backend в `app/`;
- встроенный SPA в `app/static/`;
- SQLite-хранилище через `app/db.py`;
- два набора тестов в `tests/`;
- текстовые материалы `README.md`, `main.md`, `mozg.md`.

Наблюдаемая проблема: часть русскоязычных текстов отображается в терминале как mojibake, поэтому локальный Codex-контекст сейчас частично деградирован.

## Proposed Approach

### 1. Normalize Existing Text Files

- проверить ключевые `.md` и конфигурационные файлы;
- переписать или пересохранить тексты в нормальном UTF-8;
- сделать `README.md` кратким и практичным, ориентированным на локальный запуск.

### 2. Add Codex-Focused Local Instructions

- создать `AGENTS.md` с локальными правилами работы по этому репозиторию;
- описать безопасный edit flow, команды запуска, тесты, важные каталоги и ограничения.

### 3. Add Compact Codex Documentation

Создать `docs/codex/` со следующими файлами:

- `project-overview.md` — назначение проекта и текущие возможности;
- `architecture.md` — карта модулей и поток данных;
- `workflow.md` — setup, run, test, verification;
- `backlog-notes.md` — ограничения MVP и логичные следующие шаги;
- `file-map.md` — навигатор по ключевым файлам;
- опционально `commands.md` при наличии ценности.

### 4. Add Lightweight Local Tooling

- `.editorconfig` для базовой консистентности;
- `.gitattributes` для текстовых файлов и кодировок;
- `scripts/verify.ps1` для локального прогона тестов и быстрых проверок, если не будет выглядеть лишним.

### 5. Align Repository With User Fork

- обновить `origin` на `https://github.com/lamantinX/perenoska`;
- отразить корректный URL репозитория в документации и локальном workflow;
- не трогать историю коммитов и не выполнять destructive git-команды.

## File Plan

### Files to update

- `README.md`
- `pyproject.toml`
- `main.md` (если файл нужен проекту и требует нормализации)
- `mozg.md` (если файл нужен проекту и требует нормализации)

### Files to create

- `AGENTS.md`
- `.editorconfig`
- `.gitattributes`
- `docs/codex/project-overview.md`
- `docs/codex/architecture.md`
- `docs/codex/workflow.md`
- `docs/codex/backlog-notes.md`
- `docs/codex/file-map.md`
- `docs/codex/commands.md` (опционально)
- `scripts/verify.ps1` (опционально)
- локальная git-конфигурация репозитория (`origin` remote)

## Content Principles

- Документы должны быть короткими и прикладными.
- `README.md` остается входной точкой для человека.
- `docs/codex/` дополняет `README.md`, а не дублирует его.
- Инструкции ориентированы на локальную работу пользователя через Codex.
- Изменения не должны менять поведение приложения.

## Verification Plan

После внесения изменений:

- проверить читаемость и наличие всех новых файлов;
- выполнить `pytest`;
- при необходимости проверить импорт приложения или базовую команду запуска.

## Risks

- реальное содержимое некоторых `.md` файлов может быть уже повреждено, и тогда нормализация потребует частичной ручной переписи;
- в Windows-консоли возможны ложные симптомы кодировки, поэтому нужно отличать проблемы отображения от проблем файла на диске;
- нельзя раздувать репозиторий лишней документацией, иначе Codex-контекст станет шумным.
