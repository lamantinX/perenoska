# Testing

## Test framework and execution

- Test runner: `pytest`
- Config source: `pyproject.toml`
- Main command: `python -m pytest`
- Secondary verification helper: `powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`

## Test suite structure

- `tests/test_auth_and_connections.py`
  - smoke-covers `/`
  - register/login/me flow
  - connection save/list
  - catalog `limit` pass-through
- `tests/test_transfer_preview.py`
  - end-to-end preview/import/sync flows
  - WB and Ozon payload building
  - category auto-match and manual mapping
  - dictionary issue generation and resolution
  - pagination behavior for WB client
  - various preview edge cases
- `tests/test_dictionary_mappings.py`
  - DB persistence/upsert/read behavior for category and dictionary mappings
- `tests/test_ozon_client.py`
  - Ozon pagination
  - dictionary values endpoint integration shape
  - detail enrichment fallback behavior

## Testing style

- Primary style is functional/integration-style testing with real service wiring.
- The app factory `create_app()` is used heavily to construct isolated app instances.
- Test doubles are hand-written fake clients implementing `MarketplaceClient` or subclassing concrete clients.
- Assertions focus on payload shape and user-visible API behavior more than internal call counts, except for pagination/cache probes.

## Fixtures and setup patterns

- `tmp_path` is used to isolate SQLite DB files.
- Settings are often instantiated explicitly in tests for full control over base URLs and DB path.
- Auth headers are generated by registering a user and reusing the returned token.
- Marketplace client overrides are registered through `app.state.container.client_factory.register_override(...)`.

## Coverage themes that appear strong

- Auth happy path and basic connection persistence.
- Transfer preview/import orchestration.
- Marketplace-specific payload normalization.
- Manual mapping persistence and retrieval.
- Ozon and WB pagination edge behavior.

## Coverage gaps visible from inspected tests

- No explicit tests were inspected for frontend behavior in `app/static/app.js`.
- No direct tests for `scripts/verify.ps1`.
- Limited evidence of failure-path coverage for SQLite exceptions outside auth duplicate email.
- No load/concurrency tests despite preview using `asyncio.gather()` and semaphore throttling.
- No contract tests against real WB/Ozon sandboxes in this repo snapshot.

## Verification expectations from repo instructions

- Re-read changed docs and support files.
- Run `python -m pytest`.
- Run `powershell -ExecutionPolicy Bypass -File scripts/verify.ps1`.
- Check `git status --short --branch`.
- If git setup changed, also verify `git remote -v`.
