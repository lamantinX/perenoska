## Когда валидировать

После любого изменения в документе ревью:

- Создание → [create-review.md](./create-review.md)
- После запуска `/review` (добавлена итерация)
- Перед переводом Plan Dev → DONE: статус должен быть `RESOLVED`
- При code review

## Шаги

### Шаг 0: Автоматическая валидация

```bash
python specs/.instructions/.scripts/validate-analysis-review.py {путь}
```

Скрипт проверяет все правила RV001–RV015. Если валидация пройдена — **готово**, шаги 1–6 не нужны.

**Если скрипт недоступен** — выполнить шаги 1–6 вручную.

### Шаг 1: Расположение и именование

**SSOT:** [standard-review.md § 2](./standard-review.md#2-расположение-и-именование)

- Файл расположен в `specs/analysis/NNNN-{topic}/review.md`
- Папка соответствует regex: `^\d{4}-.+$`
- `NNNN` — четырёхзначный номер (0001, 0002, ...)
- `{topic}` — kebab-case, латиница
- Имя файла — `review.md` (фиксированное)

### Шаг 2: Frontmatter

**SSOT:** [standard-review.md § 3](./standard-review.md#3-frontmatter)

**Проверить обязательные поля:**

| Поле | Проверка |
|------|----------|
| `description` | Присутствует, до 1024 символов |
| `standard` | = `specs/.instructions/analysis/review/standard-review.md` |
| `standard-version` | Присутствует, формат `vX.Y` |
| `parent` | Присутствует, путь к `plan-dev.md` ветки |
| `index` | = `specs/analysis/README.md` |
| `milestone` | Присутствует, формат `vX.Y` |
| `status` | `OPEN` или `RESOLVED` — других значений нет |

**Управление статусами:** [`chain_status.py`](../../.scripts/chain_status.py) — SSOT-модуль для переходов статусов analysis chain.

**Проверить запрещённые поля:**

| Поле | Проверка |
|------|----------|
| `children` | НЕ должен присутствовать |
| `iteration` | НЕ должен присутствовать |

### Шаг 3: Заголовок документа

**SSOT:** [standard-review.md § 2](./standard-review.md#2-расположение-и-именование)

- Формат: `# review: NNNN {Тема}`
- `NNNN` в заголовке совпадает с `NNNN` в имени папки
- Присутствуют строки `**Ветка:**` и `**Base:**`

### Шаг 4: Контекст ревью

**SSOT:** [standard-review.md § 5.1](./standard-review.md#51-контекст-ревью-заполняется-при-review-create)

Раздел `## Контекст ревью` обязателен:

| Подраздел | Проверка |
|-----------|----------|
| `### Постановка` | Присутствует. Таблица с 4 строками (discussion, design, plan-test, plan-dev) |
| `### {svc} (critical-{level})` | Хотя бы один блок сервиса |
| `### Системная документация` | Присутствует |
| `### Процесс разработки` | Присутствует |
| `### Tech-стандарты` | Присутствует |

### Шаг 5: Итерации

**SSOT:** [standard-review.md § 5.2](./standard-review.md#52-итерация-n-заполняется-при-review)

Если документ заполнен (есть итерации):
- Хотя бы одна секция `## Итерация N`
- Нумерация итераций последовательная (1, 2, 3...), без дублей
- Каждая итерация содержит:
  - Блоки `### {svc}` (по сервисам из Контекста ревью)
  - `### Интеграция (INT-N)` — один блок на итерацию
  - `### Итого` — сводная таблица
  - `**Вердикт:**` — строка с одним из: `READY`, `NOT READY`, `CONFLICT`
- Нумерация `RV-N` начинается с `RV-1` в каждой итерации

### Шаг 6: Статус и вердикт

**SSOT:** [standard-review.md § 4](./standard-review.md#4-переходы-статусов)

| Статус | Условие |
|--------|---------|
| `OPEN` | Итераций нет, ИЛИ итерация с наибольшим N содержит `NOT READY` или `CONFLICT` |
| `RESOLVED` | Итерация с наибольшим N содержит `**Вердикт:** READY` |

**Рассинхрон:** `RESOLVED` при вердикте `NOT READY`/`CONFLICT` (или наоборот) — ошибка RV013.

## Чек-лист

### Именование
- [ ] Файл в `specs/analysis/NNNN-{topic}/review.md`
- [ ] Папка: `NNNN-{topic}` (regex `^\d{4}-.+$`)
- [ ] Заголовок: `# review: NNNN {Тема}`
- [ ] NNNN в папке = NNNN в заголовке

### Frontmatter
- [ ] `description` — до 1024 символов
- [ ] `standard` = `specs/.instructions/analysis/review/standard-review.md`
- [ ] `standard-version` — формат `vX.Y`
- [ ] `parent` — путь к `plan-dev.md`
- [ ] `index` = `specs/analysis/README.md`
- [ ] `milestone` — указан, формат `vX.Y`
- [ ] `status` — `OPEN` или `RESOLVED`
- [ ] `children` — отсутствует
- [ ] `iteration` — отсутствует

### Контекст ревью
- [ ] `## Контекст ревью` присутствует
- [ ] `### Постановка` — таблица с 4 строками
- [ ] Хотя бы один `### {svc} (critical-{level})`
- [ ] `### Системная документация` присутствует
- [ ] `### Процесс разработки` присутствует
- [ ] `### Tech-стандарты` присутствует

### Итерации
- [ ] Хотя бы одна `## Итерация N` (если `status: RESOLVED`)
- [ ] Нумерация итераций без дублей
- [ ] Каждая итерация: блоки `### {svc}`, `### Интеграция (INT-N)`, `### Итого`, `**Вердикт:**`
- [ ] RV-N начинается с RV-1 в каждой итерации

### Статус
- [ ] `RESOLVED` ↔ вердикт `READY` в итерации с наибольшим N
- [ ] `OPEN` ↔ нет итераций ИЛИ последний вердикт NOT READY/CONFLICT

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Неверное расположение файла | RV001 | Файл не в `specs/analysis/NNNN-{topic}/review.md` | Переместить файл |
| Отсутствует `description` | RV002 | Пустой или отсутствующий frontmatter | Добавить description |
| Неверный `standard` | RV003 | Путь не указывает на standard-review.md | Исправить путь |
| Невалидный `status` | RV004 | Статус не `OPEN` и не `RESOLVED` | Использовать `OPEN` или `RESOLVED` |
| Отсутствует `parent` | RV005 | `parent` не указан в frontmatter | Добавить путь к plan-dev.md |
| Присутствует `children` | RV006 | review.md не имеет дочерних документов | Удалить поле `children` |
| Отсутствует `## Контекст ревью` | RV007 | Обязательный раздел пропущен | Добавить раздел |
| Нет блока `### Постановка` | RV008 | Подраздел отсутствует в Контексте ревью | Добавить `### Постановка` |
| Нет сервисного блока | RV009 | Нет ни одного `### {svc} (critical-{level})` | Добавить блок сервиса |
| Нет `## Итерация N` при RESOLVED | RV010 | `status: RESOLVED`, но итераций нет | Запустить `/review` или исправить статус |
| Дублирующийся номер итерации | RV011 | Две итерации с одинаковым N | Исправить нумерацию |
| Нет `**Вердикт:**` в итерации | RV012 | Строка вердикта отсутствует | Добавить строку вердикта |
| Рассинхрон статуса и вердикта | RV013 | `RESOLVED` при NOT READY/CONFLICT, или `OPEN` при READY | Синхронизировать статус с вердиктом итерации с max N |
| Отсутствует `milestone` | RV014 | Milestone не указан в frontmatter | Добавить milestone |
| Description слишком длинное | RV015 | Description > 1024 символов | Сократить description |
