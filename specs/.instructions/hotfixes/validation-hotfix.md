## Когда валидировать

**Обязательно:**
- После создания hotfix.md (шаг 1 TaskList)
- После завершения хотфикса (статус RESOLVED)
- При code review

**Опционально:**
- После каждого шага TaskList (промежуточная проверка)

## Шаги

### Шаг 1: Проверить frontmatter

| Проверка | Код | Критерий |
|----------|-----|----------|
| Наличие frontmatter | H001 | Файл начинается с `---` |
| Поле `date` | H002 | Формат `YYYY-MM-DD`, валидная дата |
| Поле `topic` | H003 | kebab-case, не пустое |
| Поле `status` | H004 | Одно из: `INVESTIGATING`, `IN_PROGRESS`, `RESOLVED` |
| Поле `services` | H005 | Непустой список, каждый элемент — строка |
| Поле `rollback_commit` | H006 | `—` (если статус INVESTIGATING/IN_PROGRESS до шага 5) или хеш коммита |
| Поле `rollback_command` | H007 | `—` или `git revert <hash>` |

### Шаг 2: Проверить именование файла

| Проверка | Код | Критерий |
|----------|-----|----------|
| Формат имени | H010 | `{YYYY-MM-DD}-{topic}.md`, дата совпадает с `frontmatter.date` |
| Расположение | H011 | Файл в `specs/hotfixes/` |

### Шаг 3: Проверить обязательные секции

| Проверка | Код | Критерий |
|----------|-----|----------|
| Секция "Проблема" | H020 | Заголовок `## Проблема` присутствует |
| Секция "Диагностика" | H021 | Заголовок `## Диагностика` присутствует |
| Секция "Корневая причина" | H022 | Заголовок `## Корневая причина` присутствует |
| Секция "Варианты решения" | H023 | Заголовок `## Варианты решения` присутствует |
| Секция "Выбранное решение" | H024 | Заголовок `## Выбранное решение` присутствует |
| Секция "Затронутые сервисы" | H025 | Заголовок `## Затронутые сервисы` присутствует |
| Секция "Затронутые документы" | H026 | Заголовок `## Затронутые документы` присутствует |
| Секция "Внесённые изменения" | H027 | Заголовок `## Внесённые изменения` присутствует |
| Секция "Статус" | H028 | Заголовок `## Статус` присутствует |

### Шаг 4: Проверить заполненность секций (зависит от статуса)

| Статус | Секции должны быть заполнены |
|--------|------------------------------|
| `INVESTIGATING` | 1 (Проблема) — обязательно. 2-3 (Диагностика, Корневая причина) — если шаг 2 пройден. 4-5 (Варианты, Решение) — если шаг 3 пройден |
| `IN_PROGRESS` | 1-7 — обязательно. 8 — если шаг 5 пройден |
| `RESOLVED` | 1-9 — все обязательно. Все чек-боксы в секции 9 отмечены |

### Шаг 5: Проверить форматы таблиц

| Проверка | Код | Критерий |
|----------|-----|----------|
| Варианты: code impact | H030 | Каждый вариант содержит таблицу `Файл | Что изменить | Риски` |
| Выбранное решение: план | H031 | Таблица `Файл | Что изменить | Риски` |
| Затронутые сервисы | H032 | Таблица `Сервис | Что меняется` |
| Затронутые документы: analysis | H033 | Таблица `Документ | Статус | Секции | Что обновить` |
| Затронутые документы: docs | H034 | Таблица `Документ | Секции | Что обновить` |
| Внесённые изменения: коммиты | H035 | Таблица `Хеш | Описание | Сервис` |

### Шаг 6: Проверить согласованность

| Проверка | Код | Критерий |
|----------|-----|----------|
| status vs чек-лист | H040 | Если RESOLVED — все чек-боксы `[x]` |
| status vs rollback | H041 | Если RESOLVED или IN_PROGRESS (шаг 5+) — `rollback_commit` ≠ `—` |
| services vs секция 6 | H042 | Список сервисов в frontmatter совпадает с таблицей "Затронутые сервисы" |
| Минимум 2 варианта | H043 | Секция 4 содержит `### Вариант 1` и `### Вариант 2` |

## Чек-лист

- [ ] Frontmatter валиден (H001-H007)
- [ ] Именование файла корректно (H010-H011)
- [ ] Все 9 секций присутствуют (H020-H028)
- [ ] Заполненность соответствует статусу (шаг 4)
- [ ] Форматы таблиц корректны (H030-H035)
- [ ] Согласованность данных (H040-H043)

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Нет frontmatter | H001 | Файл создан вручную | Добавить YAML frontmatter по шаблону |
| Дата не совпадает | H010 | Имя файла и frontmatter.date различаются | Синхронизировать |
| rollback_commit = `—` при RESOLVED | H041 | Забыли заполнить на шаге 5 | Заполнить хеш коммита HEAD до изменений |
| Один вариант решения | H043 | Нет альтернативы | Добавить вариант "Без изменений" с рисками |
| Не все чек-боксы при RESOLVED | H040 | Преждевременный перевод в RESOLVED | Завершить все шаги или вернуть IN_PROGRESS |
