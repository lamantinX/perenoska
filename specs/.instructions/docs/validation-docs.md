## Когда валидировать

Автоматически — при каждом коммите (pre-commit hook).

Вручную — после изменения структуры `specs/docs/`:

- Добавление или удаление сервисного документа
- Добавление или удаление per-tech стандарта
- Изменение структуры `.system/` или `.technologies/`

## Шаги

### Шаг 0: Автоматическая валидация

```bash
python specs/.instructions/.scripts/validate-docs.py
```

Скрипт проверяет все правила DOC001-DOC006. Если валидация пройдена — **готово**, шаги 1-5 не нужны.

**Если скрипт недоступен** — выполнить шаги 1-5 вручную.

### Шаг 1: Проверить директории

Проверить наличие обязательных директорий:

| Директория | Назначение |
|-----------|-----------|
| `specs/docs/` | Корневая папка документации |
| `specs/docs/.system/` | Системные документы |
| `specs/docs/.technologies/` | Per-tech стандарты |

### Шаг 2: Проверить системные документы

Проверить наличие обязательных файлов в `.system/`:

| Файл | Назначение |
|------|-----------|
| `specs/docs/.system/overview.md` | Архитектура системы |
| `specs/docs/.system/conventions.md` | Конвенции API и shared-интерфейсы |
| `specs/docs/.system/infrastructure.md` | Платформа и окружения |
| `specs/docs/.system/testing.md` | Стратегия тестирования |

### Шаг 3: Проверить индекс

Проверить наличие `specs/docs/README.md` — индекс сервисов и навигация.

### Шаг 4: Проверить файлы-примеры

Проверить наличие файлов-примеров:

| Файл | Назначение |
|------|-----------|
| `specs/docs/example.md` | Пример сервисного документа |
| `specs/docs/.technologies/standard-example.md` | Пример per-tech стандарта |

### Шаг 5: Проверить шаблонные per-tech стандарты

Проверить наличие обязательных шаблонных per-tech стандартов (создаются при init):

| Файл | Назначение |
|------|-----------|
| `specs/docs/.technologies/standard-openapi.md` | Конвенции OpenAPI 3.1 — REST-контракты |
| `specs/docs/.technologies/standard-protobuf.md` | Конвенции Protobuf v3 — gRPC-контракты |
| `specs/docs/.technologies/standard-asyncapi.md` | Конвенции AsyncAPI 3.0 — event-схемы |

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Отсутствует директория docs/ | DOC001 | Не инициализирована структура docs/ | Создать `specs/docs/` и обязательные поддиректории |
| Отсутствует директория .system/ | DOC002 | Не создана папка системных документов | Создать `specs/docs/.system/` |
| Отсутствует директория .technologies/ | DOC002 | Не создана папка per-tech стандартов | Создать `specs/docs/.technologies/` |
| Отсутствует обязательный системный документ | DOC003 | Файл не создан или удалён | Создать недостающий файл по per-document стандарту |
| Отсутствует README.md | DOC004 | Индекс не создан | Создать `specs/docs/README.md` по стандарту |
| Отсутствует файл-пример | DOC005 | Пример не создан | Создать `example.md` или `standard-example.md` |
| Отсутствует шаблонный per-tech стандарт | DOC006 | Шаблон не создан при init | Создать недостающий `standard-{tech}.md` по стандарту |
