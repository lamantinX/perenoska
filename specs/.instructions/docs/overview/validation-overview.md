## Когда валидировать

Автоматически — при каждом коммите (pre-commit hook), если изменены файлы в `specs/docs/.system/`.

Вручную — после:

- Добавления или удаления сервиса
- Изменения связей между сервисами
- Обновления сквозных потоков
- Изменения контекстной карты доменов
- Обновления shared-кода

## Шаги

### Шаг 0: Автоматическая валидация

```bash
python specs/.instructions/.scripts/validate-docs-overview.py
```

Скрипт проверяет все правила OVW001-OVW011. Если валидация пройдена — **готово**, шаги 1-8 не нужны.

**Если скрипт недоступен** — выполнить шаги 1-8 вручную.

### Шаг 1: Проверить frontmatter

Проверить наличие обязательных полей:

| Поле | Значение |
|------|---------|
| `description` | Непустая строка |
| `standard` | `specs/.instructions/docs/overview/standard-overview.md` |

### Шаг 2: Проверить секции

Проверить наличие всех 6 обязательных h2-секций в правильном порядке:

| # | Секция | Формат |
|---|--------|--------|
| 1 | Назначение системы | `## Назначение системы` |
| 2 | Карта сервисов | `## Карта сервисов` |
| 3 | Связи между сервисами | `## Связи между сервисами` |
| 4 | Сквозные потоки | `## Сквозные потоки` |
| 5 | Контекстная карта доменов | `## Контекстная карта доменов` |
| 6 | Shared-код | `## Shared-код` |

Дополнительно проверить: заголовок h1 `# Архитектура системы`.

### Шаг 3: Проверить таблицы

Проверить наличие таблиц в секциях и корректность колонок:

| Секция | Обязательные колонки |
|--------|---------------------|
| Карта сервисов | Сервис, Зона ответственности, Критичность, Владеет данными, Ключевые API |
| Связи между сервисами | Источник, Приёмник, Протокол, Назначение, Паттерн |
| Контекстная карта доменов | Домен, Реализует сервис, Агрегаты, Связь с другими доменами |
| Shared-код | Пакет, Назначение, Владелец, Потребители |

**Порядок строк:** таблица Карты сервисов — алфавитный по «Сервис». Таблица Связей — алфавитный по «Источник», при равенстве — по «Приёмник». Таблица Контекстной карты — алфавитный по «Домен».

**Значения колонки «Критичность»:** каждая строка таблицы Карты сервисов должна содержать в колонке «Критичность» одно из допустимых значений: `critical-high`, `critical-medium`, `critical-low`. Другие значения — ошибка.

### Шаг 4: Проверить mermaid-схему

Проверить в секции «Карта сервисов»:

1. Наличие code-блока ` ```mermaid `
2. Mermaid содержит `subgraph` (группировка по ролям обязательна)
3. Mermaid содержит хотя бы один узел и связь

### Шаг 5: Проверить сквозные потоки

Проверить:

1. Секция «Сквозные потоки» содержит 2-5 h3-подсекций (`### {Название}`)
2. Каждый поток содержит строку `**Участники:**`
3. Каждый поток содержит code-блок с шагами
4. Каждый поток содержит раздел `**Ключевые контракты:**` с минимум 1 ссылкой

### Шаг 6: Проверить консистентность сервисов

**Основная проверка.** Множества сервисов должны совпадать между секциями:

1. Собрать сервисы из колонки «Сервис» таблицы Карты сервисов → **Set A**
2. Собрать уникальные сервисы из колонок «Источник» и «Приёмник» таблицы Связей (исключая внешние системы) → **Set B**
3. Собрать сервисы из колонки «Реализует сервис» таблицы Контекстной карты → **Set C**
4. Проверить: **Set A** = **Set B** = **Set C**

**Допустимые исключения для Set B:** внешние системы (отсутствуют в Карте сервисов и Контекстной карте), акторы (`frontend`, `admin frontend`).

### Шаг 7: Проверить DDD-паттерны

Проверить наличие в секции «Контекстная карта доменов» одного из подразделов:

- **DDD-паттерны связей** — если проект использует DDD
- **Связи между доменами** — если проект НЕ использует DDD (альтернативный формат)

Если секция содержит stub-текст (`*...*`) — проверка пропускается.

### Шаг 8: Проверить вводные абзацы

Проверить, что каждая из 5 секций (Карта сервисов, Связи, Сквозные потоки, Контекстная карта, Shared-код) начинается с абзаца свободного текста **перед** таблицей, mermaid-блоком или h3-подсекцией.

Секции со stub-текстом (`*...*`) — пропускаются.

## Чек-лист

- [ ] Frontmatter: `description` и `standard` присутствуют
- [ ] Заголовок h1 `# Архитектура системы`
- [ ] 6 обязательных h2-секций в правильном порядке
- [ ] Таблица Карты сервисов с 5 колонками (включая Критичность)
- [ ] Значения Критичности — только `critical-high`, `critical-medium`, `critical-low`
- [ ] Mermaid-блок в секции Карта сервисов
- [ ] Таблица Связей с 5 колонками
- [ ] Хотя бы 1 сквозной поток с Участниками, шагами, контрактами
- [ ] Таблица Контекстной карты с 4 колонками
- [ ] Таблица Shared-кода с 4 колонками (или заглушка)
- [ ] Консистентность: сервисы в Карте = Связях = Контекстной карте
- [ ] Алфавитный порядок строк в таблицах (Связи: двухуровневый — Источник → Приёмник)
- [ ] DDD-паттерны связей или Связи между доменами в Контекстной карте
- [ ] Вводные абзацы перед таблицами во всех секциях

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Отсутствует или некорректный frontmatter | OVW001 | Нет YAML-блока или отсутствует `description`/`standard` | Добавить frontmatter по шаблону |
| Отсутствует обязательная секция | OVW002 | Пропущена одна из 6 h2-секций | Добавить секцию по шаблону из standard-overview.md |
| Секции в неправильном порядке | OVW003 | h2-секции не соответствуют фиксированному порядку | Переупорядочить секции |
| Таблица не содержит обязательных колонок | OVW004 | Пропущена колонка или опечатка в заголовке | Привести колонки в соответствие стандарту |
| Mermaid-схема отсутствует или некорректна | OVW005 | Нет блока, нет subgraph, нет узлов или связей | Добавить mermaid с subgraph и связями |
| Сквозной поток некорректен | OVW006 | Нет Участников, шагов, контрактов; количество не 2-5; нет ссылок | Дополнить поток по шаблону |
| Консистентность сервисов нарушена | OVW007 | Сервис есть в одной секции, но отсутствует в другой | Синхронизировать списки сервисов между секциями |
| Нарушен алфавитный порядок | OVW008 | Строки таблицы не отсортированы (Связи: двухуровневая сортировка) | Пересортировать таблицу |
| Отсутствует подраздел DDD-паттернов | OVW009 | Нет «DDD-паттерны связей» и нет «Связи между доменами» | Добавить подраздел по стандарту |
| Отсутствует вводный абзац | OVW010 | Секция начинается с таблицы/mermaid/h3 без вводного текста | Добавить 1-3 предложения перед содержимым |
| Некорректное значение критичности | OVW011 | Колонка «Критичность» отсутствует или содержит недопустимое значение | Указать одно из: `critical-high`, `critical-medium`, `critical-low` |
