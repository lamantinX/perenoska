## Когда валидировать

**Обязательно:**
- После создания или изменения `specs/docs/.system/conventions.md`
- Перед коммитом (автоматически через pre-commit hook)
- При code review

**Опционально:**
- При обновлении shared-пакета (проверить секцию Shared-пакеты)
- При изменении {svc}.md (проверить что ссылается на conventions.md, а не дублирует)

## Коды ошибок

| Код | Описание | Критичность |
|-----|----------|-------------|
| CNV001 | Отсутствует или некорректный frontmatter | Блокирующая |
| CNV002 | Отсутствует обязательная секция | Блокирующая |
| CNV003 | Секции в неправильном порядке | Блокирующая |
| CNV004 | Таблица не содержит обязательных колонок | Предупреждение |
| CNV005 | Отсутствует code-блок с реализацией | Предупреждение |
| CNV006 | Shared-пакет без обязательного элемента | Предупреждение |
| CNV007 | Пустая обязательная секция без stub-текста | Предупреждение |
| CNV008 | Секция «Требования по уровням критичности» отсутствует или неполная | Блокирующая |

## Шаги

### Шаг 0: Автоматическая проверка

```bash
python specs/.instructions/.scripts/validate-docs-conventions.py
```

Если скрипт пройден — валидация завершена. Если есть ошибки — исправить по кодам ниже.

### Шаг 1: Проверить frontmatter

**CNV001:** frontmatter должен содержать:
- `description` — непустое описание
- `standard` — путь к `specs/.instructions/docs/conventions/standard-conventions.md`

### Шаг 2: Проверить обязательные секции

**CNV002:** Проверить наличие всех 8 обязательных h2-секций:
1. `## Формат ответов`
2. `## Формат ошибок`
3. `## Пагинация`
4. `## Аутентификация`
5. `## Версионирование API`
6. `## Shared-пакеты`
7. `## Логирование`
8. `## Требования по уровням критичности`

**CNV003:** Первые 8 секций — в фиксированном порядке. Дополнительные секции допустимы после «Требования по уровням критичности».

### Шаг 3: Проверить таблицы

**CNV004:** Проверить обязательные колонки в таблицах:

| Секция | Обязательные колонки |
|--------|---------------------|
| Формат ошибок | HTTP Status, Код, Когда |
| Пагинация | Параметр, Тип, Default, Max, Описание |
| Логирование (уровни) | Уровень, Когда использовать, Пример |
| Логирование (поля) | Поле, Источник, Описание |

### Шаг 4: Проверить code-блоки

**CNV005:** Проверить наличие code-блоков с реализацией в секциях:
- Формат ошибок — code-блок с базовым классом ошибки
- Пагинация — code-блок с базовыми классами
- Логирование — code-блок с базовым паттерном

Code-блоки должны содержать рабочий код на языке проекта (не псевдокод, не пустые placeholder-ы).

### Шаг 5: Проверить shared-пакеты

**CNV006:** Для каждого h3 в секции Shared-пакеты проверить наличие 4 обязательных элементов:
1. `**Владелец:**` — непустое значение
2. Code-блок с интерфейсом (сигнатуры)
3. Таблица (параметры/поля/функции)
4. Code-блок с примером использования

### Шаг 6: Проверить пустые секции

**CNV007:** Если секция Shared-пакеты не содержит h3-подсекций — должен быть stub-текст в курсиве (`*...*`). Пустая секция без stub-текста — ошибка.

### Шаг 7: Проверить требования по уровням критичности

**CNV008:** Секция `## Требования по уровням критичности` должна содержать:
1. Вводный абзац с пояснением уровней и ссылкой на `{svc}.md`
2. Таблица отказоустойчивости с колонками: Критерий, critical-high, critical-medium, critical-low
3. Таблица логирования с колонками: Критерий, critical-high, critical-medium, critical-low

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Нет frontmatter | CNV001 | Файл начинается без `---` | Добавить frontmatter по шаблону |
| Секция «Пагинация» отсутствует | CNV002 | Пропущена при создании | Добавить секцию с таблицей и реализацией |
| Секции не в порядке | CNV003 | Логирование перед Shared-пакетами | Переставить секции по стандарту |
| Нет колонки «Паттерн» | CNV004 | Таблица скопирована из старого формата | Добавить недостающую колонку |
| Псевдокод вместо реализации | CNV005 | Написан абстрактный пример | Заменить на рабочий code-блок |
| Shared-пакет без примера | CNV006 | Забыли добавить пример использования | Добавить code-блок с рабочим примером |
