## Когда валидировать

**Обязательно:**
- После создания или изменения `specs/docs/.system/infrastructure.md`
- Перед коммитом (автоматически через pre-commit hook)
- При code review

**Опционально:**
- При добавлении нового сервиса (проверить секции Сервисы и порты, Service Discovery)
- При добавлении нового хранилища (проверить секцию Хранилища)

## Шаги

### Шаг 0: Автоматическая проверка

```bash
python specs/.instructions/.scripts/validate-docs-infrastructure.py
```

Если скрипт пройден — валидация завершена. Если есть ошибки — исправить по кодам ниже.

### Шаг 1: Проверить frontmatter

**INF001:** frontmatter должен содержать:
- `description` — непустое описание
- `standard` — путь к `specs/.instructions/docs/infrastructure/standard-infrastructure.md`

### Шаг 2: Проверить обязательные секции

**INF002:** Проверить наличие всех 8 обязательных h2-секций:
1. `## Локальный запуск`
2. `## Сервисы и порты`
3. `## Хранилища`
4. `## Message Broker`
5. `## Service Discovery`
6. `## Окружения`
7. `## Мониторинг и логи`
8. `## Секреты`

**INF003:** Первые 8 секций — в фиксированном порядке. Дополнительные секции допустимы после Секретов.

### Шаг 3: Проверить таблицы

**INF004:** Проверить обязательные колонки в таблицах:

| Секция | Обязательные колонки |
|--------|---------------------|
| Сервисы и порты | Сервис, Хост (docker), Хост (local), Порт, URL, Healthcheck |
| Хранилища (каждое) | Параметр, Значение (dev), Env-переменная |
| Service Discovery | Сервис-источник, Сервис-цель, Механизм, Значение (dev), Env-переменная |
| Окружения | Параметр, dev, staging, prod |
| Мониторинг (где смотреть) | Окружение, Логи, Метрики |
| Секреты | Секрет, Env-переменная, Источник (dev), Источник (prod) |

### Шаг 4: Проверить хранилища

**INF005:** Для каждого h3 в секции Хранилища проверить наличие 2 обязательных элементов:
1. Таблица параметров (Хост, Порт, База, Пользователь, Пароль)
2. Connection string (`**Connection string:**`)

### Шаг 5: Проверить окружения

**INF006:** Таблица Окружений должна содержать обязательные строки:
- Реплики сервисов
- БД
- Секреты
- Домен
- Логирование

### Шаг 6: Проверить пустые секции

**INF007:** Если секция не содержит h3-подсекций и не содержит таблиц — должен быть stub-текст в курсиве (`*...*`). Пустая секция без stub-текста — ошибка.

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Нет frontmatter | INF001 | Файл начинается без `---` | Добавить frontmatter по шаблону |
| Секция «Message Broker» отсутствует | INF002 | Пропущена при создании | Добавить секцию (stub-текст если брокера нет) |
| Секции не в порядке | INF003 | Окружения перед Service Discovery | Переставить секции по стандарту |
| Нет колонки «Healthcheck» | INF004 | Таблица сервисов неполная | Добавить недостающую колонку |
| Хранилище без connection string | INF005 | Забыли добавить шаблон подключения | Добавить `**Connection string:**` |
| Нет строки «Домен» в Окружениях | INF006 | Обязательная строка пропущена | Добавить строку в таблицу |
