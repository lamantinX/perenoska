## Когда валидировать

**Обязательно:**
- После создания или изменения любого `specs/docs/{svc}.md`
- Перед коммитом (автоматически через pre-commit hook)
- При code review

**Опционально:**
- При добавлении нового endpoint-а (проверить секцию API контракты)
- При изменении Data Model (проверить секцию Data Model)
- При изменении зависимостей (проверить секцию Зависимости)

|-----|----------|-------------|
| SVC001 | Отсутствует или некорректный frontmatter | Блокирующая |
| SVC002 | Отсутствует обязательная секция | Блокирующая |

## Шаги

### Шаг 0: Автоматическая проверка

```bash
python specs/.instructions/.scripts/validate-docs-service.py
```

Если скрипт пройден — валидация завершена. Если есть ошибки — исправить по кодам ниже.

### Шаг 1: Проверить frontmatter

**SVC001:** frontmatter должен содержать:
- `description` — непустое описание
- `standard` — путь к `specs/.instructions/docs/service/standard-service.md`

**SVC010:** frontmatter должен содержать:
- `criticality` — одно из допустимых значений: `critical-high`, `critical-medium`, `critical-low`

### Шаг 2: Проверить обязательные секции

**SVC002:** Проверить наличие всех 10 обязательных h2-секций:
1. `## Назначение`
2. `## API контракты`
3. `## Data Model`
4. `## Потоки`
5. `## Code Map`
6. `## Зависимости`
7. `## Доменная модель`
8. `## Границы автономии LLM`
9. `## Planned Changes`
10. `## Changelog`

**SVC003:** Секции должны идти в указанном порядке. Дополнительные секции не допускаются.

### Шаг 3: Проверить таблицы

**SVC004:** Проверить обязательные колонки в таблицах (если секция не stub):

| Секция / Подсекция | Обязательные колонки |
|---------------------|---------------------|
| Code Map → Tech Stack | Технология, Версия, Назначение, Стандарт |
| Code Map → Пакеты | Пакет, Назначение, Ключевые модули |
| Доменная модель → Агрегаты | Агрегат, Описание |
| Доменная модель → Доменные события | Событие, Описание |
| Code Map → Makefile таргеты | Таргет, Команда, Описание |

Таблицы в API контрактах и Data Model проверяются в шагах 4 и 5 (зависят от типа endpoint-а / хранилища).

### Шаг 4: Проверить подсекции API контрактов

**SVC005:** Если секция API контракты не stub — проверить:
1. Есть хотя бы одна h3-подсекция (endpoint, WebSocket или Event)
2. Каждый REST endpoint содержит: `**Auth:**`, `**Паттерн:**`, `**Errors:**`
3. Каждый REST endpoint содержит `**Response`

### Шаг 5: Проверить подсекции Data Model

**SVC006:** Если секция Data Model не stub — проверить:
1. Есть хотя бы одна h3-подсекция (таблица / коллекция / Redis ключ)
2. PostgreSQL/SQL таблица содержит колонки: Колонка, Тип, Constraints, Описание
3. Redis ключ содержит колонки: Ключ, Тип, TTL, Описание
4. MongoDB коллекция содержит колонки: Поле, Тип, Required, Описание

### Шаг 6: Проверить границы автономии LLM

**SVC007:** Секция «Границы автономии LLM» должна содержать все три уровня:
- `**Свободно:**` — что можно менять без согласования
- `**Флаг:**` — что требует проверки
- `**CONFLICT:**` — что нельзя менять без обсуждения

### Шаг 7: Проверить Changelog

**SVC008:** Секция «Changelog» должна содержать минимум одну запись. Первая запись — о создании сервиса (формат: `**Создание сервиса**` или ссылка на `analysis/`).

### Шаг 8: Проверить пустые секции

**SVC009:** Если секция не содержит таблиц, code-блоков, h3-подсекций или bold-текста — должен быть stub-текст в курсиве (`*...*`). Пустая секция без stub-текста — ошибка.

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Нет frontmatter | SVC001 | Файл начинается без `---` | Добавить frontmatter по шаблону |
| Секция «Доменная модель» отсутствует | SVC002 | Пропущена при создании | Добавить секцию (stub-текст если инфраструктурный сервис) |
| Секции не в порядке | SVC003 | Code Map перед Потоками | Переставить секции по стандарту |
| Нет или некорректное `criticality` | SVC010 | Поле отсутствует или значение не из списка | Добавить `criticality: critical-high \| critical-medium \| critical-low` |
| Нет колонки «Constraints» в Data Model | SVC004 | Таблица неполная | Добавить недостающую колонку |
| Endpoint без Auth и Errors | SVC005 | Неполное описание | Добавить обязательные элементы |
| Data Model без h3-подсекции | SVC006 | Только текст без таблицы | Добавить подсекцию с таблицей колонок |
| Нет уровня CONFLICT | SVC007 | Пропущен при заполнении | Добавить все три уровня |
| Пустой Changelog | SVC008 | Забыли при создании | Добавить запись о создании сервиса |
