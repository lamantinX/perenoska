## Когда валидировать

**Обязательно:**
- После создания или изменения `specs/docs/.system/testing.md`
- Перед коммитом (автоматически через pre-commit hook)
- При code review

**Опционально:**
- При добавлении нового типа тестов (проверить секции Типы тестов, Команды запуска)
- При добавлении/удалении сервиса (проверить секции Структура файлов, Межсервисные тесты)

## Коды ошибок

| Код | Описание | Критичность |
|-----|----------|-------------|
| TST001 | Отсутствует или некорректный frontmatter | Блокирующая |
| TST002 | Отсутствует обязательная секция | Блокирующая |
| TST003 | Секции в неправильном порядке | Блокирующая |
| TST004 | Таблица не содержит обязательных колонок | Предупреждение |
| TST005 | Отсутствует обязательный элемент секции | Предупреждение |
| TST006 | Принципы тестовых данных неполны | Предупреждение |
| TST007 | Пустая обязательная секция без stub-текста | Предупреждение |

## Шаги

### Шаг 0: Автоматическая проверка

```bash
python specs/.instructions/.scripts/validate-docs-testing.py
```

Если скрипт пройден — валидация завершена. Если есть ошибки — исправить по кодам ниже.

### Шаг 1: Проверить frontmatter

**TST001:** frontmatter должен содержать:
- `description` — непустое описание
- `standard` — путь к `specs/.instructions/docs/testing/standard-testing.md`

### Шаг 2: Проверить обязательные секции

**TST002:** Проверить наличие всех 6 обязательных h2-секций:
1. `## Типы тестов`
2. `## Структура файлов`
3. `## Стратегия мокирования`
4. `## Межсервисные тесты`
5. `## Тестовые данные`
6. `## Команды запуска`

**TST003:** Первые 6 секций — в фиксированном порядке. Дополнительные секции допустимы после Команд запуска.

### Шаг 3: Проверить таблицы

**TST004:** Проверить обязательные колонки в таблицах:

| Секция | Обязательные колонки |
|--------|---------------------|
| Типы тестов | Тип, Что проверяет, Scope, Внешние зависимости, Когда запускается |
| Стратегия мокирования | Уровень, БД, Message Broker, Другие сервисы, Shared-код, Внешние API |

### Шаг 4: Проверить дерево файлов

**TST005:** Секция «Структура файлов» должна содержать:
1. Code-блок с деревом директорий
2. Дерево содержит `src/` и `tests/`
3. Строки дерева содержат комментарии (`#`)

### Шаг 5: Проверить принципы тестовых данных

**TST006:** Секция «Тестовые данные» должна содержать маркированный список принципов (минимум 3 пункта). Обязательные смыслы:
- Независимость тестов (ключевые слова: независим, порядок запуска, межтестов)
- Минимальность данных (ключевые слова: минимальн, фабрик, валидн)
- Изоляция данных (ключевые слова: изоляци, очища, rollback, truncate)

### Шаг 6: Проверить пустые секции

**TST007:** Если секция не содержит таблиц, code-блоков, h3-подсекций или bold-текста — должен быть stub-текст в курсиве (`*...*`). Пустая секция без stub-текста — ошибка.

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Нет frontmatter | TST001 | Файл начинается без `---` | Добавить frontmatter по шаблону |
| Секция «Межсервисные тесты» отсутствует | TST002 | Пропущена при создании | Добавить секцию (stub-текст если один сервис) |
| Секции не в порядке | TST003 | Тестовые данные перед Стратегией мокирования | Переставить секции по стандарту |
| Нет колонки «Scope» в Типах тестов | TST004 | Таблица неполная | Добавить недостающую колонку |
| Нет дерева в Структуре файлов | TST005 | Code-блок отсутствует | Добавить дерево по шаблону |
| Только 2 принципа в Тестовых данных | TST006 | Пропущен обязательный принцип | Добавить недостающий принцип |
