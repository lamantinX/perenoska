## 1. Назначение

`docs/README.md` — точка входа в контур docs/. Чистый навигатор: таблица сервисов со ссылками на `{svc}.md`, ссылки на системные документы (`.system/`) и стандарты технологий (`.technologies/`).

**Содержит:** навигацию по всем документам docs/.

**НЕ содержит:** архитектурное описание системы (это в `.system/overview.md`).

---

## 2. Секции

Документ состоит из 5 частей в фиксированном порядке.

| # | Часть | Markdown-уровень | Содержание |
|---|-------|-----------------|-----------|
| 1 | Шапка документа | h1 + frontmatter | Название системы, назначение, ссылка на overview |
| 2 | Сервисы | h2 `## Сервисы` | Таблица: сервис, назначение, технологии, ссылка |
| 3 | Системные документы | h2 `## Системные документы` | Ссылки на .system/ файлы |
| 4 | Стандарты технологий | h2 `## Стандарты технологий` | Ссылки на .technologies/ файлы |
| 5 | Дерево | h2 `## Дерево` | ASCII-дерево docs/ |

### Шапка документа

Шапка — не h2-секция, а верхняя часть файла:

1. **Frontmatter** — обязателен, содержит `description` и `standard`
2. **Заголовок h1** — `# {Название системы}`
3. **Описание** — одно предложение, назначение системы
4. **Ссылка на архитектуру** — `**Архитектура:** [overview.md](.system/overview.md)`

### Сервисы

Таблица всех сервисов проекта.

**Колонки:**

| Колонка | Содержание |
|---------|-----------|
| Сервис | Имя сервиса (совпадает с именем папки в `/src/`) |
| Назначение | Краткое описание, 5-10 слов (сокращение из секции "1. Назначение" соответствующего `{svc}.md`) |
| Технологии | Ключевые технологии: язык, БД, MQ |
| Документ | Ссылка на `{svc}.md` |

**Порядок строк:** алфавитный по имени сервиса.

**Если сервисов нет:** оставить таблицу с заголовком без строк данных.

### Системные документы

Фиксированная таблица — 4 документа из `.system/`:

| Документ | Описание |
|----------|----------|
| `overview.md` | Архитектура системы: связи сервисов, сквозные потоки, контекстная карта |
| `conventions.md` | Конвенции API: формат ошибок, пагинация, auth + shared-интерфейсы |
| `infrastructure.md` | Платформа: деплой, сети, мониторинг, окружения |
| `testing.md` | Тестирование: типы, структура, мокирование, команды |

Эта таблица не меняется — набор системных документов фиксирован.

### Стандарты технологий

Таблица всех per-tech стандартов из `.technologies/`.

**Колонки:**

| Колонка | Содержание |
|---------|-----------|
| Технология | Название технологии |
| Стандарт | Ссылка на `standard-{tech}.md` |

**Порядок строк:** алфавитный по названию технологии.

**Если per-tech стандартов нет:** оставить секцию с таблицей без строк данных и пометкой `*Per-tech стандарты добавляются при подключении технологии.*`.

### Дерево

ASCII-дерево `specs/docs/` — все файлы и директории с комментариями.

**Порядок:** сначала служебные папки (`.system/`, `.technologies/`), затем per-service файлы в алфавитном порядке, затем `README.md`.

## 3. Принципы

**Только навигация.** README.md — навигатор, не описание системы. Архитектурный контекст — в `.system/overview.md`.

**Синхронизация с файловой системой.** Таблица сервисов, таблица технологий и дерево соответствуют реальному содержимому `specs/docs/`. При добавлении/удалении сервиса или технологии — обновить все три места.

**Описания — краткие.** Колонка "Назначение" в таблице сервисов — сокращение из секции "1. Назначение" соответствующего `{svc}.md`, не дословная копия. Формат: 5-10 слов, передающих суть сервиса.
