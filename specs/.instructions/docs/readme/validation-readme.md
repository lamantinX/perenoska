## Когда валидировать

Автоматически — при каждом коммите (pre-commit hook), если изменены файлы в `specs/docs/`.

Вручную — после:

- Добавления или удаления сервисного документа (`{svc}.md`)
- Добавления или удаления per-tech стандарта (`standard-{tech}.md`)
- Изменения структуры `specs/docs/`
- Редактирования `specs/docs/README.md`

## Шаги

### Шаг 0: Автоматическая валидация

```bash
python specs/.instructions/.scripts/validate-docs-readme.py
```

Скрипт проверяет все правила RDM001-RDM006. Если валидация пройдена — **готово**, шаги 1-5 не нужны.

**Если скрипт недоступен** — выполнить шаги 1-5 вручную.

### Шаг 1: Проверить frontmatter

Проверить наличие обязательных полей:

| Поле | Значение |
|------|---------|
| `description` | Непустая строка |
| `standard` | `specs/.instructions/docs/readme/standard-readme.md` |

### Шаг 2: Проверить секции

Проверить наличие всех обязательных h2-секций в правильном порядке:

| # | Секция | Формат |
|---|--------|--------|
| 1 | Сервисы | `## Сервисы` |
| 2 | Системные документы | `## Системные документы` |
| 3 | Стандарты технологий | `## Стандарты технологий` |
| 4 | Дерево | `## Дерево` |

Дополнительно проверить шапку: заголовок h1 и строка `**Архитектура:**`.

### Шаг 3: Проверить дерево

Основная проверка — дерево в секции `## Дерево` соответствует реальной файловой системе `specs/docs/`:

1. Каждый файл/папка из дерева существует в файловой системе
2. Каждый файл/папка из файловой системы присутствует в дереве
3. Порядок: `.system/` и `.technologies/` первыми, затем per-service файлы в алфавитном порядке, `README.md` последним

**Исключения из проверки:** файлы `example.md` и `standard-example.md` — примеры, не обязательны в дереве.

### Шаг 4: Проверить таблицу сервисов

Проверить соответствие таблицы `## Сервисы` и per-service файлов:

1. Каждый `{svc}.md` в `specs/docs/` (кроме `example.md`) имеет строку в таблице
2. Каждая строка таблицы ссылается на существующий `{svc}.md`
3. Строки в алфавитном порядке

### Шаг 5: Проверить таблицу технологий

Проверить соответствие таблицы `## Стандарты технологий` и per-tech файлов:

1. Каждый `standard-{tech}.md` в `specs/docs/.technologies/` (кроме `standard-example.md`) имеет строку в таблице
2. Каждая строка таблицы ссылается на существующий файл
3. Строки в алфавитном порядке

## Типичные ошибки

| Ошибка | Код | Причина | Решение |
|--------|-----|---------|---------|
| Отсутствует frontmatter | RDM001 | Нет YAML-блока в начале файла | Добавить frontmatter с `description` и `standard` |
| Отсутствует обязательная секция | RDM002 | Пропущена одна из 4 h2-секций | Добавить секцию по шаблону из standard-readme.md |
| Дерево не соответствует файловой системе | RDM003 | Файл добавлен/удалён, дерево не обновлено | Синхронизировать дерево с `specs/docs/` |
| Таблица сервисов рассинхронизирована | RDM004 | Сервис добавлен/удалён, таблица не обновлена | Добавить/удалить строку в таблице |
| Таблица технологий рассинхронизирована | RDM005 | Per-tech файл добавлен/удалён | Добавить/удалить строку в таблице |
| Нарушен алфавитный порядок | RDM006 | Строки не отсортированы | Пересортировать таблицу/дерево |
