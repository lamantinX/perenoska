# 0002: Корректный перенос карточек товаров WB↔Ozon

## Проблема / Контекст

**Текущее состояние (AS IS):**
Сервис Perenoska реализует базовый перенос карточек товаров между Wildberries и Ozon через API обоих маркетплейсов. Клиенты `WBClient` и `OzonClient` реализуют общий ABC `MarketplaceClient` с методами `get_products()`, `create_products()` и другими. `TransferService` оркестрирует перенос: preview → launch → sync. Маппинг категорий частично реализован через `MappingService`.

Однако структуры данных WB и Ozon существенно различаются:
- **Категории:** WB использует собственный справочник субъектов (`/content/v2/object/parent/list`), Ozon — независимое дерево категорий (`/v1/description-category/tree`). Атрибуты категорий на каждом маркетплейсе уникальны и периодически меняются.
- **Бренды:** WB возвращает бренд как строку. Ozon требует валидного `brand_id` из своего справочника (`/v1/brand/list`). WB принимает бренд как строку (без верификации по справочнику).
- **Описание:** WB хранит описание в поле `description`. Ozon принимает его в поле `annotation` при создании карточки.
- **Фотографии:** WB возвращает список URL фотографий. Ozon принимает массив URL при создании — URL должны быть публично доступны.
- **Атрибуты:** Каждая категория на WB и Ozon имеет свой набор обязательных и опциональных атрибутов — их надо получать динамически через API справочников.

**Проблема:**
Перенос карточек работает некорректно из-за несоответствия структур данных: поля не маппируются, категории не сопоставляются, бренды не верифицируются, фото и описания передаются в неверных полях. Пользователь не может успешно перенести товар с WB на Ozon (приоритетное направление) без ручной доработки карточки после переноса.

**Вне scope текущего Milestone:**
Товары с вариантами (размерная сетка, цвета) имеют разные модели данных на WB и Ozon — перенос вариантов вынесен в следующий Milestone. Текущая реализация переносит один товар как одну карточку.

## Критерии успеха

- Перенос карточки WB→Ozon: поля `name`, `vendor_code`, `description→annotation`, `brand` (`brand_id`), `category_id`, `images`, атрибуты корректно заполнены в созданной карточке Ozon в статусе черновика
- Перенос карточки Ozon→WB: поля `name`, `vendor_code`, `annotation→description`, `brand` (строка), категория, `images`, атрибуты корректно заполнены в созданной карточке WB в статусе черновика
- Категории: система получает справочник категорий через API (не хардкод), предлагает категорию через LLM, пользователь может изменить; при ненайденной — предлагает выбор из дерева
- Бренды (WB→Ozon): система ищет `brand_id` через API Ozon с нечётким поиском; при ненайденном — уведомление пользователю
- Фотографии: URL фотографий из источника передаются в целевую карточку; при недоступности — preview блокирует перенос с предупреждением
- Обязательные атрибуты: при автоматическом несопоставлении — запрос ручного ввода пользователя; перенос без обязательных атрибутов не запускается
- 100% юнит-тестов для маппинга полей, категорий (mock LLM) и брендов проходят
- Интеграционные тесты с реальными API: карточка успешно создаётся на целевом маркетплейсе
- Preview завершается менее чем за 10 секунд; перенос одной карточки — менее чем за 30 секунд
- Товары с вариантами (размерная сетка, цвета) — вне scope текущего Milestone
